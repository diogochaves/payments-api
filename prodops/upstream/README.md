# Upstream

Upstream is the lightweight ProdOps path for exploration before delivery
commitment.

Use it for experiments, spikes, prototypes, hypothesis validation, BDD
exploration, OBC preparation, contract investigation, and Reliability Plan
enrichment.

Upstream output should be reversible and should either:

- produce a learning;
- close an uncertainty;
- create or refine a Downstream candidate;
- update a relevant ProdOps artifact when evidence supports it.

Record activity in `upstream-trail.md`.
