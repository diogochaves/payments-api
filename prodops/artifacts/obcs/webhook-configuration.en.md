# OBC - Webhook Configuration

## Status

Downstream. Status `Entrou` in `prodops/artifacts/plans/iteration-plan.md` (section "Iteration Plan recomendado").

## Business Outcome

Magazine Siará can register a webhook URL associated with its API token and
receive automatic notifications on every relevant payment status change,
without needing to actively poll the Payments API. The system signs each delivery
with HMAC-SHA256 so the receiver can verify authenticity. Local environments
work without additional certificate configuration.

## Observable Events

| Event | Meaning | Required dimensions |
| --- | --- | --- |
| `webhook.registered` | New webhook was registered for a token. | `tenantId`, `tokenId`, `webhookId`, `url` (masked), `events[]`, `correlationId` |
| `webhook.deleted` | Webhook was removed. | `tenantId`, `tokenId`, `webhookId`, `correlationId` |
| `webhook.delivery.sent` | HTTP delivery was dispatched successfully (2xx). | `tenantId`, `webhookId`, `event`, `deliveryId`, `durationMs` |
| `webhook.delivery.failed` | HTTP delivery failed or returned non-2xx status. | `tenantId`, `webhookId`, `event`, `deliveryId`, `statusCode`, `reason` |

## Domain Events Delivered to Webhooks

| Internal event | External payload name | Trigger |
| --- | --- | --- |
| `payment.confirmed` | `invoice.confirmed` | Payment confirmed by the provider |
| `payment.cancelled` | `invoice.cancelled` | Invoice cancelled |

## Webhook Payload Contract

```json
{
  "deliveryId": "uuid-v4",
  "webhookId": "wh_xxx",
  "event": "invoice.confirmed",
  "tenantId": "magazine-siara",
  "timestamp": "2026-07-03T10:00:00.000Z",
  "payload": {
    "invoiceId": "...",
    "orderId": "...",
    "providerPaymentId": "...",
    "amount": 159.90,
    "currency": "BRL",
    "status": "CONFIRMED",
    "confirmedAt": "2026-07-03T10:00:00.000Z"
  }
}
```

Headers on delivery:
- `X-Payments-Signature: sha256=<hmac-sha256 of the body with the webhook secret>`
- `X-Payments-Delivery-Id: <deliveryId>`
- `Content-Type: application/json`

## Initial SLIs

| SLI | Initial target |
| --- | --- |
| Webhooks registered with a valid URL return `201` with `webhookId` and `secret`. | 100% |
| `secret` appears only in the creation response; never in listing or logs. | 100% |
| Deliveries for `invoice.confirmed` dispatched within 5 seconds of the event. | 95% |
| Delivery failure does not block the payment confirmation flow. | 100% |
| `webhook.delivery.failed` emitted for every delivery with non-2xx status or timeout. | 100% |

## Reliability Rules

- Maximum of 10 active webhooks per `tokenId`.
- URL must be `https://` in any environment, except `localhost`/`127.0.0.1` which accept `http://`.
- The `secret` field is generated by the system at creation and cannot be changed — only recreated with a new webhook.
- Delivery failure is recorded observably but does not cause automatic retry in this first slice.
- Webhook does not interfere with the critical confirmation path: delivery is dispatched in the background (fire-and-forget).
- Webhook URL is masked in logs (domain only, without the full path).

## Data Model — DynamoDB WebhooksTable

### Main table

```
PK:  TOKEN#{tokenId}
SK:  WEBHOOK#{webhookId}
```

### GSI1 — lookup by tenant (used in delivery)

```
GSI1PK: TENANT#{tenantId}
GSI1SK: WEBHOOK#{webhookId}
```

### Attributes

| Attribute | Type | Description |
| --- | --- | --- |
| `webhookId` | String | UUID v4 generated at creation |
| `tokenId` | String | Token that registered the webhook |
| `tenantId` | String | Token's tenant |
| `url` | String | Delivery destination URL |
| `events` | StringSet | E.g.: `["invoice.confirmed","invoice.cancelled"]` |
| `secret` | String | HMAC key generated at creation; not exposed after creation |
| `description` | String? | Optional label |
| `active` | Boolean | `true` while the webhook is active |
| `createdAt` | String | ISO 8601 |
| `updatedAt` | String | ISO 8601 |

### Model rationale

- **PK by `tokenId`**: listing query by token is O(1) without scan.
- **GSI1 by `tenantId`**: delivery service queries by tenant without knowing the raw token.
- **`events` as StringSet**: efficient filtering in DynamoDB without an additional GSI.
- **`secret` separate from URL**: allows URL change by re-registering; revocation is by deletion.
- **No TTL in this slice**: indefinite retention; TTL for inactive items can be added in a later iteration.
