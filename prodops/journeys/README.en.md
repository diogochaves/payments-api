# Journeys

The ProdOps Framework has five journeys.

## Classic Journeys

| Journey | Purpose |
|---|---|
| [Discovery](discovery/) | Exploration, experiments, hypothesis validation |
| [Delivery](delivery/) | Implementation, tests, deploy and promotion |
| [Operation](operation/) | Operation, incidents, runbooks, postmortems |

## Cross-cutting Journeys

| Journey | Purpose |
|---|---|
| [Assessment](assessment/) | Continuous evaluation of risks, opportunities and plans |
| [Diligence](diligence/) | Continuous observation, triggers, verifications and coordination |

Assessment and Diligence continuously observe the other journeys. They do not represent only documentation — they represent active Framework behavior.

## Execution Mode

Journeys can be executed in two modes:

- **Upstream** — exploration, low commitment
- **Downstream** — governed delivery, mandatory criteria

→ [Execution Model](../execution-model/README.md)
